Von Thomas Groten Dorst:

A new version of CafRunner / CafPlayer is available with improvements mainly for usability:

- CafRunner introduces a "new" type of file which contains configuration data for CafRunner *.caf. By double clicking on such files, CafRunner is started and the configured algo is executed.
Caf-files are nearly the same as the well know "*.ini" files - except that they contain some additional information like the path to the algo-dll(s) and some options when running in Player-mode.
The installation of the new functionality is done in the background when starting CafRunner in the current way. The new functionality is an add-on to the existing methods of configuration.
CafRunner (Menu "File / save configuration") provides a command for generating a .caf file for the algorithm currently running in CafRunner. This has an option which allows you to create a snapshot of all the configuration settings with all the parameters of your algorithm, so that no further configuration-files (like e.g. vehicle.ini) are needed. This snapshot may be usefull when testing an algo with different configuration settings.
This snapshot is also saved when doing a measurement in the car. In this special case CafRunner adapts some settings to prepare it for using it in Replay-mode. So in the end with double-clicking on the saved <algo>.caf within a measurement you can easily start the algo with the same conditions like in the car and directly see its behaviour in the replay of the measurement  (probably you have to adjust the path to your local copy of the algo.dll first).

- CafPlayer stores the last six measurements used as input for the replay (Menu "File"). So it is easier to work with several measurements. Another benefit: If you start CafPlayer it automatically preselects the last used measurement, so you can start the replay with only one click.


A more detailed list of changes can be seen in the "? / About" or in the help-file "cafrunner.chm / Versionhistory".

The new version is - as always - automatically distributed via SVN.
