Hello colleagues,
 
 
 
we wanted to give you an update from the tooling side regarding your workflow / Linux development.
 
We have three new information that we want to share with you:
 
 
 
1. GRID BUILD without DOCKER
 
We have now provided all necessary libraries as debian packages to build the GRID for Linux x86 platform.
 
These are combined in a meta-package, so that the installation is easier.
 
Thus, it is no longer necessary to use the docker container if you want to build the GRID under Linux x86.
 
Cross-compile and driveworks still need the docker containers.
 
 
 
Information about the meta-package can be found here: http://github.conti.de/CTZS/phad_ubuntu_migration/wiki/Using-PHAD-Debian-Repository
 
 
 
 
 
2. GUI = ON for Linux Pipelines
 
In the software structure meeting, we determined that the <ProjectName>_CMake_Pipeline_Linux build jobs in Jenkins would be set to GUI = ON.
 
This change gives you the possibility to migrate the visualizations of GRID to Linux.
 
The introduced GIT workflow remains untouched and will continue to be built with GUI = OFF.
 
If all projects are prepared for GUI=ON build on Linux we can switch to GUI=ON for the GIT-Workflow.
 
 
 
Here is a direct link to the job overview: http://jenkinsctzs.conti.de/view/00_CMake_Pipeline/
 
 
 
 
 
3. Cross-Platform IDE
 
We can now offer C-Lion licenses.
 
C-Lion is a cross-platform IDE from JetBrains.
 
Kerstin Keller has agreed to give a small introductory training (more information on this in the near future).
 
 
 
The C-Lion licenses are limited to a number of 70, so if you are not sure if you want to use the IDE, you can also use the trial version.
 
A trial version can be downloaded here: https://www.jetbrains.com/clion/download/#section=linux
 
 
 
To get a C-Lion license and to install the IDE, please do the following:
 
 
 
Please create a JetBrains Account!
 
Always use your Continental E-Mail address, DON'T use your private one.
 
 
 
Account information should be your Firstname, Surname and as Username your Continental E-Mail address.
 
 
 
Create Account here: https://account.jetbrains.com/login
 
Click the following link to accept this invitation and start using your CLion license: https://account.jetbrains.com/a/li3qu2jf
 
 
 
All other necessary information will be provided during the installation.
 If you want to use CLion on Windows and Linux, please use the activation key for offline usage

Dear colleagues,
 
 
 
we updated the available images from eu-test.artifactory.conti.de.
 
To update your local images to the new releases you need to do the following steps:
 
 
 
Caution, all data in the container will be deleted, so safe all files you need.
 
 
 
1. Delete all current docker containers
 
 
 
Check if ALL docker containers are exited:                  docker ps -a
 
IF a docker is running or pending:                                  docker stop <ID>
 
Remove all docker container:                                         docker rm -f $(docker ps -aq)
 
 alte Mail zu Docker
 
2. Update the Docker_Get_Started-Scritps:
 
 
 
We also updated the Docker_Get_Started-Scripts.
 
Please remove your checked out Docker_Get_Started.
 
Clone the origin repo on your linux machine from: http://github.conti.de/CTZS/Docker_Get_Started.git
 
 
 
3. Execute the Docker-Image-Update-Script:
 
 
 
Change the directory:                cd /your/copy/of/Docker_Get_Started/docker-scripts/
 
Make the script executable:       chmod +x docker_image_update.sh
 
Execute the script:                    ./docker_image_update.sh
 
 
 
Improvements and Errors
 
 
 
In order to improve the docker images and the work with them, we ask you to report any errors,
 
missing programs, missing libraries, etc. as an issue at the following address: http://github.conti.de/CTZS/Docker_Get_Started/issues
