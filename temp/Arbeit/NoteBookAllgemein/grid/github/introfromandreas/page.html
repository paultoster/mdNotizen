The workflow with GIT is a little different than that with SVN.
Firstly you might want to install some tools to be able to work with GIT:

Windows:
GIT for Windows, the actual GIT Program: https://GIThub.com/GIT-for-windows/GIT/releases/download/v2.16.2.windows.1/GIT-2.16.2-64-bit.exe
Tortoise GIT, GUI client similar to tortoise SVN: https://download.tortoiseGIT.org/tGIT/2.6.0.0/TortoiseGIT-2.6.0.0-64bit.msi
Other available GUI clients are SmartGit (via SSP7), GITHub Desktop (via Browser) or have a look here:

Ubuntu:
Use the command to install GIT: sudo apt-get install git
GUI client options:
https://wiki.ubuntuusers.de/Grafische_Oberfl%C3%A4chen_f%C3%BCr_Git/
https://git-scm.com/download/gui/linux
--------------------------------------------------------------------------------
It's important to understand the basic and not so basic concepts of GIT when working with it effectively.
We recommend these two web tutorials:
https://try.GIThub.io  -> Deals in general with commiting and the workflow on the console (GIT status, GIT add, GIT commit).
https://learnGITbranching.js.org/ -> web tutorial, explains very well and visually the concepts of branching, merging & rebasing.
Even if you've used GIT before, give the second tutorial a try, it's highly educational!
